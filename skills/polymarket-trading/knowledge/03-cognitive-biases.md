- **Annie Duke**《Thinking in Bets: Making Smarter Decisions When You Don't Have All the Facts》

## 為什麼認知偏誤是交易者的頭號敵人

> **「聰明人更容易自欺欺人，因為他們知道認知偏誤的存在後，反而會過度自信。」**
> — Annie Duke

認知偏誤是系統性的思維錯誤，它們：
- 不會隨著經驗自動消失
- 對專家和新手同樣有效
- 在壓力下更容易發生
- 能被識別但難以完全避免

**預測市場的殘酷現實**：你的對手可能也知道這些偏誤，並在利用你的偏誤賺錢。

## 七大核心偏誤

### 1. Resulting 偏誤（結果論）

#### 定義
用結果的好壞來判斷決策的品質。

```
❌ Resulting 思維：
   結果好 → 決策正確
   結果差 → 決策錯誤

✅ 正確思維：
   好決策 + 好運氣 → 好結果
   好決策 + 壞運氣 → 壞結果
   壞決策 + 好運氣 → 好結果（最危險！）
   壞決策 + 壞運氣 → 壞結果
```

#### 在預測市場的表現

| 場景 | Resulting 反應 | 正確反應 |
|------|----------------|----------|
| 你估計 70%，買了 YES，結果沒發生 | "我判斷錯了" | "30% 的機率本來就會發生" |
| 你估計 30%，沒買，結果發生了 | "我錯過了" | "70% 的時候我是對的" |
| 你估計 90%，大倉位買入，贏了 | "我真厲害" | "這次對了，但過程對嗎？" |

#### 防禦方法

**1. 決策日誌分離法**
```markdown
## 決策記錄（結果前）
- 市場：____
- 我的估計：____%
- 市場價格：____%
- 理由（3 條）：
  1.
  2.
  3.
- 倉位大小：____%

## 結果記錄（結果後）
- 實際結果：____
- Brier Score：____

## 過程評估（獨立於結果）
- [ ] 理由在決策時是合理的嗎？
- [ ] 倉位大小符合 Kelly 嗎？
- [ ] 有新信息我忽略了嗎？
- [ ] 如果重來 100 次，我會做同樣的決定嗎？
```

**2. 心理實驗：想像 100 次**
> 如果我做這個決定 100 次，長期會賺還是賠？

**3. 慶祝過程，不是結果**
- ✅ "這個分析做得很徹底"
- ❌ "這筆賺錢了"

### 2. 過度自信偏誤（Overconfidence Bias）

#### 定義
高估自己判斷的準確性，給出過窄的預測區間。

#### Kahneman 的發現
```
專家被問 "90% 信心區間" 時：
- 他們認為只有 10% 的結果會落在區間外
- 實際上有 40-50% 的結果落在區間外
```

#### 在預測市場的表現

| 過度自信信號 | 現實 |
|-------------|------|
| "這個肯定會發生" | 沒有 100% 的事 |
| "我比市場更了解這個" | 市場是集體智慧 |
| "我的記錄一直很好" | 可能只是運氣 |
| 連續 3 次贏後加大倉位 | 小樣本謬誤 |

#### 校準測試

**快速自我檢測**：
```
給自己 10 個問題，每個都給 90% 信心的答案
如果你校準良好，應該有 9 個對
大多數人只對 5-7 個
```

#### 防禦方法

**1. 強制使用分數 Kelly**
```python
# 不管多有信心，永遠用分數 Kelly
def safe_kelly(kelly_raw, max_fraction=0.5):
    return min(kelly_raw * max_fraction, 0.10)  # 單筆最多 10%
```

**2. 區間思維**
```
❌ "我認為概率是 70%"
✅ "我認為概率在 60-80%，中點 70%"
```

**3. Pre-Mortem 練習**
> 假設你的預測錯了。回溯：最可能的原因是什麼？

**4. 追蹤校準度**
- 記錄所有預測
- 計算 Brier Score
- 目標：< 0.15

### 3. 確認偏誤（Confirmation Bias）

#### 定義
選擇性地尋找、解讀和記憶支持自己觀點的信息。

#### Kahneman 的解釋
> **「當人們首先相信了一個錯誤的陳述，他們極有可能相信支持它的論點。這是確認偏誤的基礎。」**

#### 在預測市場的表現

**你買了 YES 之後**：
- ✅ 你會注意到支持 YES 的新聞
- ❌ 你會忽略或淡化反對 YES 的新聞
- ✅ 你會認為市場價格上漲是"理性反應"
- ❌ 你會認為市場價格下跌是"噪聲"

#### 防禦方法

**1. 主動尋找反證**
```markdown
## 我的觀點：____

## 反對我的觀點的 3 個最強論點：
1.
2.
3.

## 這些反對意見的反駁：
1.
2.
3.

## 調整後的觀點：____
```

**2. 紅隊思考**
> 如果有人專門想證明我錯了，他們會怎麼說？

**3. 信息源多樣化**
- 主動閱讀對立觀點
- 關注那些過去預測準確但與你意見不同的人
- 避免回音室

**4. 先預測，後看市場**
```
正確順序：
1. 寫下你的概率估計
2. 記錄理由
3. 然後才看市場價格

錯誤順序：
1. 看市場價格
2. 找理由說市場錯了
3. 下注
```

### 4. 錨定效應（Anchoring Effect）

#### 定義
過度依賴最先接收到的信息（"錨"），即使它與決策無關。

#### Kahneman 的經典實驗
```
A 組：先問 "聯合國中非洲國家比例是否超過 65%？"
       再問 "你認為比例是多少？"
       平均回答：45%

B 組：先問 "聯合國中非洲國家比例是否超過 10%？"
       再問 "你認為比例是多少？"
       平均回答：25%

實際答案：約 28%
```

#### 在預測市場的表現

**常見錨點**：
- 市場當前價格（最危險的錨）
- 昨天的價格
- 你上次交易的價格
- 某個專家說的數字
- 新聞標題中的數字

**錨定如何影響你**：
```
市場價格：40¢
你的獨立估計可能是：60%

但因為先看到 40¢：
→ 你的估計被拉向 40%
→ 最終你可能說 50%
→ Edge 被低估，該交易的沒交易
```

#### 防禦方法

**1. 盲估計法**
```markdown
## 市場：____

### 第一步：蒙眼估計（不看市場價格）
我的獨立估計：____%
理由：
-
-

### 第二步：查看市場價格
市場價格：____%

### 第三步：調整
差異：±____%
我的估計需要調整嗎？為什麼？
最終估計：____%
```

**2. 多錨點技術**
> 不要只依賴一個參考點。收集 3-5 個不同來源的估計，然後綜合。

**3. 從 Base Rate 開始**
```
錯誤：從市場價格開始調整
正確：從歷史 base rate 開始，根據具體情況調整
```

### 5. 可得性偏誤（Availability Heuristic）

#### 定義
根據事例容易想起的程度來判斷事件的可能性。

#### Kahneman 的解釋
> **「可得性偏誤發生在我們用一個顯著事件、近期經歷或特別生動的事情來做判斷時。」**

#### 在預測市場的表現

| 可得性陷阱 | 現實 |
|-----------|------|
| 最近看到類似事件發生 → 高估概率 | 一次不代表趨勢 |
| 媒體大量報導某風險 → 高估該風險 | 媒體追求眼球，不追求準確 |
| 沒聽說過類似事件 → 低估概率 | 不常見 ≠ 不可能 |
| 記得上次大贏 → 高估勝率 | 選擇性記憶 |

**實例**：
```
9/11 後：
- 人們大幅高估恐怖襲擊概率
- 許多人改開車而不是坐飛機
- 結果：開車的事故死亡 > 9/11 死亡人數
```

#### 防禦方法

**1. 永遠查 Base Rate**
```
問自己：
"不考慮最近發生的事，歷史上這種情況發生的頻率是多少？"
```

**2. 數據優先**
```
❌ "我記得上次..." → 做決策
✅ 查找歷史數據 → 計算頻率 → 做決策
```

**3. 冷卻期**
> 重大新聞後，等 24-48 小時再做交易決策。

**4. 可得性審計**
```markdown
## 我的估計：____%

## 為什麼我這麼估計？
- 理由 1：____
- 理由 2：____

## 這些理由是基於數據還是印象？
- 理由 1：數據 / 印象
- 理由 2：數據 / 印象

## 如果基於印象，需要調整嗎？
```

### 6. 後見之明偏誤（Hindsight Bias）

#### 定義
事件發生後，認為"我早就知道會這樣"的傾向。

#### Tetlock 的研究
> **「後見之明偏誤讓我們無法從錯誤中學習，因為我們重寫了記憶，讓自己看起來比實際更聰明。」**

#### 在預測市場的表現

**典型表現**：
```
事件發生前：你認為 40%
事件發生後：你說"我一直覺得很可能"

事件發生前：你認為 60%
事件沒發生：你說"我一直有懷疑"
```

**為什麼危險**：
- 阻止你識別錯誤的預測模式
- 讓你無法改進
- 增加過度自信

#### 防禦方法

**1. 寫下來，不能改**
```markdown
## 預測記錄（結果前）
日期：____
市場：____
我的估計：____%
理由：____

[結果出來後不得修改上述內容]

## 結果記錄
結果：____
我當時的預測準確嗎？對比上面的記錄
```

**2. 定期回顧**
- 每週回顧本週的預測
- 計算準確率和 Brier Score
- 誠實面對錯誤

**3. 承認不確定性**
> 用概率語言。"我認為有 60% 的概率"比"我覺得會發生"更好，因為它承認了 40% 的錯誤可能。

### 7. 沉沒成本謬誤（Sunk Cost Fallacy）

#### 定義
因為已經投入（時間、金錢、精力）而繼續一個明顯應該放棄的行動。

#### 在預測市場的表現

**典型場景**：
```
你以 60¢ 買了 YES
市場跌到 40¢
新信息顯示你的估計應該改為 35%

沉沒成本思維：
"我已經虧了 20¢，不能現在賣，要等回本"

正確思維：
"我現在的估計是 35%，市場是 40%，應該賣出或做空"
```

**為什麼特別危險**：
- 虧損會觸發情緒反應
- "回本"心理很難克服
- 可能導致加倉攤平（更大的錯誤）

#### 防禦方法

**1. 零基礎測試**
```
問自己：
"如果我今天沒有持倉，看到這個價格，我會買嗎？"

如果答案是"不會"→ 你應該賣出
```

**2. 每日重新評估**
```markdown
## 每日持倉檢查

### 市場 A
- 當前持倉：____
- 當前價格：____
- 我現在的概率估計：____
- 如果是新交易，我會做嗎？____
- 決定：持有 / 減倉 / 清倉 / 加倉
```

**3. 預設退出規則**
```
入場時就設定：
- 如果我的估計變化超過 ±15%，重新評估
- 如果市場價格變化超過 ±20%，檢查原因
- 如果出現特定事件 X，立即退出
```

**4. 心理技巧**
> 假裝持倉是別人的，你只是在幫他管理。你會建議他怎麼做？

## 偏誤檢測清單

### 交易前清單
```markdown
## 偏誤自檢

### 1. 錨定效應
- [ ] 我是先獨立估計，還是先看市場價格？
- [ ] 我的估計受到什麼數字影響？

### 2. 確認偏誤
- [ ] 我有主動尋找反對我的證據嗎？
- [ ] 我能列出 3 個反對理由嗎？

### 3. 可得性偏誤
- [ ] 我的估計基於數據還是印象？
- [ ] 有近期事件影響我的判斷嗎？

### 4. 過度自信
- [ ] 我給的是範圍還是點估計？
- [ ] 我的倉位符合 Kelly 原則嗎？

### 通過 ✅ → 可以交易
有任何 ❌ → 重新評估或不交易
```

### 交易後清單
```markdown
## 結果分析

### 1. Resulting 檢查
- [ ] 我是用結果還是過程來評估這筆交易？
- [ ] 如果重來 100 次，我會做同樣的決定嗎？

### 2. 後見之明檢查
- [ ] 我的記錄和實際預測一致嗎？
- [ ] 我有沒有說"我早就知道"？

### 3. 沉沒成本檢查
- [ ] 如果沒有這筆持倉，我現在會進場嗎？
- [ ] 我的持倉決定是基於現在的估計還是過去的成本？
```

## 系統性防禦策略

### 1. 建立預測記錄系統

**最低要求**：
- 記錄每個預測及理由
- 追蹤結果
- 計算 Brier Score

**進階**：
- 分析偏誤模式
- 建立個人校準曲線
- 定期回顧和調整

### 2. 使用決策日誌

見 `04-trading-log-template.md`

### 3. 建立"求真小組"

**Annie Duke 的建議**：
- 找 2-3 個也做預測市場的人
- 規則：獎勵改變想法，懲罰 resulting
- 定期分享預測（不是結果）讓他人挑戰

### 4. 定期偏誤審計

**每月回顧**：
```markdown
## 本月偏誤審計

### 數據
- 總預測數：____
- 平均 Brier Score：____
- 預測 > 70% 的實際發生率：____%
- 預測 < 30% 的實際發生率：____%

### 發現的偏誤模式
1.
2.

### 下月改進計劃
1.
2.
```
