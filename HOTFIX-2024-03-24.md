# 🔧 Hotfix - 2024-03-24

## 問題描述

Bot 在執行工具（特別是 WebFetch、Bash 等）時崩潰，錯誤訊息：
```
Claude Code process exited with code 1
```

## 根本原因

`src/session.ts` 中硬編碼了原作者的路徑：
```typescript
/Users/heddaai/clawd/piggyx/MEMORY.md
```

當 bot 在其他系統運行時：
1. ❌ 路徑不存在
2. ❌ 安全檢查拒絕訪問
3. ❌ Claude Code 進程崩潰

## 修復內容

### 1. 移除硬編碼路徑
- ❌ 刪除 `/Users/heddaai/clawd/piggyx/MEMORY.md`
- ✅ 改為使用環境變量 `MEMORY_FILE_PATH`

### 2. Memory 加載變成可選
- 如果設置了 `MEMORY_FILE_PATH` 且文件存在，則自動加載
- 如果沒有設置或文件不存在，則跳過（不會崩潰）

### 3. 更新配置文件
在 `.env.example` 中添加：
```bash
# 记忆文件路径 (可选)
MEMORY_FILE_PATH=
```

## 如何更新

### 方法 1：拉取最新代碼（推薦）

```bash
cd hedda-telegram-bot
git pull origin main
```

### 方法 2：手動修改

如果無法 git pull，手動編輯 `src/session.ts`：

找到這段代碼（約第 236-243 行）：
```typescript
// Auto-load memory for piggyx
const memoryPrefix = `🧠 載入記憶中...\n\n請讀取核心記憶檔案：
/Users/heddaai/clawd/piggyx/MEMORY.md
...
```

替換為：
```typescript
// Optional: Auto-load memory if MEMORY.md exists in project
let memoryPrefix = "";
const memoryPath = process.env.MEMORY_FILE_PATH;
if (memoryPath) {
  try {
    const memoryFile = Bun.file(memoryPath);
    if (await memoryFile.exists()) {
      memoryPrefix = `🧠 Loading memory...\n\nPlease read the core memory file at:\n${memoryPath}\n\nThen proceed with the following message:\n\n`;
    }
  } catch (error) {
    console.warn(`Memory file not accessible: ${memoryPath}`);
  }
}
```

## 驗證修復

更新後重啟 bot：
```bash
# 停止舊進程
killall -9 bun

# 重新啟動
./start.sh
```

測試消息：
```
install bird cli https://github.com/jawond/bird
```

應該正常執行不會崩潰。

## 可選：使用 Memory 功能

如果你想使用 memory 功能：

1. 創建 `memory/MEMORY.md`：
```bash
cp memory/MEMORY.md.template memory/MEMORY.md
```

2. 編輯 `.env`，添加：
```bash
MEMORY_FILE_PATH=./memory/MEMORY.md
```

3. 重啟 bot
